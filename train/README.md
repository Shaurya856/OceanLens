# train/

Full training pipeline for SeabedDetector.

| File | Purpose |
|------|---------|
| `run.py` | CLI entry point — `python train/run.py --annotations ... --image-dir ...` |
| `trainer.py` | Two-phase training loop: backbone frozen → full fine-tune; saves checkpoints |
| `dataset.py` | `SeabedDataset` — loads COCO-style annotation JSON and returns tensors |
| `loss.py` | Composite detection loss: focal CE + L1 + GIoU + confidence BCE |
| `matcher.py` | Hungarian bipartite matcher for assigning predictions to ground-truth boxes |
| `augmentations.py` | Underwater-specific augmentations: flips, red-channel suppression, blur |
| `generate_annotations.py` | Builds `data/annotations.json` from a `data/images/<species>/` folder layout |

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate           # macOS / Linux
pip install -r requirements.txt
```

## Data layout

Place training images under `data/images/` with one subfolder per species:

```
data/
├── annotations.json          ← generated by generate_annotations.py
└── images/
    ├── Seahorse/
    │   ├── img001.jpg
    │   └── img002.jpg
    ├── Sharks/
    │   └── img001.jpg
    └── ...
```

## Quick start

```bash
# Step 1 — generate annotation file from image folders
python train/generate_annotations.py
# equivalent to:
python train/generate_annotations.py --dataset-dir data/images --out data/annotations.json

# Step 2 — smoke test (2 epochs, 32 samples — verifies end-to-end pipeline)
python train/run.py \
    --annotations data/annotations.json \
    --image-dir   data/images \
    --smoke-test

# Step 3 — full training run
python train/run.py \
    --annotations data/annotations.json \
    --image-dir   data/images \
    --epochs 80 \
    --warmup-epochs 10 \
    --batch-size 8 \
    --head-lr 1e-4 \
    --backbone-lr 1e-5
```

Checkpoints are written to `checkpoints/` after every epoch.  
Final weights are copied to `weights/detector.pt` at the end of training.

## CLI options (`train/run.py`)

| Flag | Default | Description |
|------|---------|-------------|
| `--annotations` | *(required)* | Path to annotation JSON |
| `--image-dir` | *(required)* | Root folder of species image subfolders |
| `--checkpoint-dir` | `checkpoints` | Where to save per-epoch checkpoints |
| `--epochs` | 80 | Total training epochs |
| `--warmup-epochs` | 10 | Backbone-frozen warmup epochs (Phase 1) |
| `--batch-size` | 8 | Batch size |
| `--head-lr` | 1e-4 | LR for neck, decoder, and heads |
| `--backbone-lr` | 1e-5 | LR for backbone during Phase 2 |
| `--weight-decay` | 1e-4 | AdamW weight decay |
| `--val-split` | 0.1 | Fraction of data held out for validation |
| `--num-workers` | 0 | DataLoader workers (use 4+ on Linux with GPU) |
| `--smoke-test` | — | Run 2 epochs on 32 samples only |

## Two-phase training

**Phase 1 (epochs 0 → warmup_epochs):** Backbone frozen. Only BiFPN, decoder, and classification heads are trained.

**Phase 2 (epochs warmup_epochs → total_epochs):** Full fine-tune. Backbone unfrozen with 10× smaller LR and gradient clipping at `max_norm=0.1`.

After Phase 2, one forward pass populates `NoveltyDetector` prototype embeddings from real training distributions.
