# inference/

Species detection pipeline — model loading, per-image inference, and async batch scheduling.

| File | Purpose |
|------|---------|
| `runner.py` | Singleton model loader (`get_model`) and `run_inference(image_bytes) -> dict` |
| `batch_processor.py` | Processes a batch of images; appends results to `results.json`; sends WS events |
| `scheduler.py` | Splits image list into batches and runs them concurrently; emits `job_done` when complete |

## Async job flow

```
POST /infer
  └─ inference/scheduler.schedule_inference()    ← asyncio.create_task
       └─ inference/batch_processor.process_inference_batch()  for each batch
            └─ inference/runner.run_inference(image_bytes)
                 └─ loop.run_in_executor()        GPU work off event loop
                      └─ WebSocket events per image; results.json written to disk
```

## Model requirements

The inference pipeline requires trained weights before detections are meaningful:

| File | Path | Notes |
|------|------|-------|
| Detector weights | `weights/detector.pt` | Produced by `python train/run.py` |
| Taxonomy labels | `weights/taxonomy_labels.json` | Auto-generated by `train/generate_annotations.py` |

Without `weights/detector.pt`, the model loads with randomly initialised heads (ImageNet backbone only) and will produce noise detections.

## Result format

`GET /infer/{job_id}/results` returns a list of per-image dicts:

```json
{
  "frame_id": "...",
  "image_id": "...",
  "filename": "frame_000001.png",
  "detections": [
    {
      "detection_id": "...",
      "bbox": {"x1": 120, "y1": 80, "x2": 340, "y2": 260},
      "confidence": 0.87,
      "taxonomy": {"phylum": "Chordata", "species": "Seahorse"},
      "is_novel": false,
      "novelty_score": 0.12
    }
  ]
}
```
